# webAppProjectOne


Web Application Development Project Assignment 
Minimum 7 interlinked HTML pages, consistent design, responsive layout, HTML/CSS/JS/Bootstrap, realistic content, Git/GitHub with specific commit 
dates and a minimum of 34 commits. 
1. Refined Assignment Instructions (Student-Facing) 
Objective 
Build a fully responsive, multi-page web application that presents realistic, well-structured content. Apply core front-end technologies (HTML, CSS, 
JavaScript, Bootstrap) while using professional version control with Git and GitHub. 
Core Requirements 
# Requirement 
Details 
1 Pages 
At least 7 distinct HTML pages. All pages must be interlinked via a clear navigation menu (header/footer) and contextual 
hyperlinks. 
2 Consistency 
3 Technologies 
Unified visual identity across all pages: same colour scheme, typography, navigation layout, footer, and overall design 
language. 
HTML5, CSS3, JavaScript (ES6+), and Bootstrap 5 (or 4). You may add custom CSS on top of Bootstrap. 
4 Responsiveness 
Pages must be rendered correctly on desktop, tablet, and mobile devices. Use Bootstrap’s grid system, flexbox, and 
media queries. 
5 Realistic Content 
All text, images, and data must reflect a real-world theme (e.g., a small business, portfolio, e-commerce showcase, 
community hub, educational platform). No placeholder “lorem ipsum” for the main content – use genuine, coherent 
information. 
6 Interactivity 
(JavaScript) 
Implement at least two interactive features, for example: a dynamic contact form with validation, a filterable product 
gallery, an image carousel, a dark/light mode toggle, or a live search/filter. 
7 Version Control (Git & 
GitHub) - Create a private or public GitHub repository. - First commit must be dated 15 June 2026. - Provide a repository link by 21 June 2026 to show incremental progress.  - Final commit must be dated 28 June 2026. - Total commits ≥ 34 by submission deadline 28 June 2026 23:59. - Commit messages must be meaningful (e.g., “Add navigation bar”, “Implement responsive product grid”). - No large “final” uploads – commits must show incremental progress. 
8 Submission 
Provide the GitHub repository URL by 21 June 2026 23:59. Ensure the repository includes all source files (HTML, CSS, 
JS, assets). The live preview (GitHub Pages or any static host) is recommended but not mandatory.

Additional Guidelines 
 Page structure example (choose your own theme): Home, About, Services/Products, Blog/News, Contact, Gallery, FAQ or Testimonials. 
 Navigation: Use a responsive Bootstrap navbar that collapses on mobile. 
 Forms: At least one functional form (e.g., contact, sign-up, comment) with JavaScript validation. 
 Images: Optimize for web; use alt attributes for accessibility. 
 Code quality: Indent properly, use meaningful class/id names, separate CSS/JS files where appropriate (but inline in a single page is allowed only 
for demonstration; external files preferred). 
 Browser compatibility: Test on latest Chrome, Firefox, Edge. 
Submission Deadline 
28 June 2026, 23:59 (local time). Late submissions will yield a Zero mark. No Make-up assessment shall be planned.
